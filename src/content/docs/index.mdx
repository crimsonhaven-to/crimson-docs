## The five halls of the castle

<CardGrid>
	<LinkCard title="The backend (the brain)" href="/self-hosting/backend/" description="Metadata, accounts, the login wall, and orchestration. Knows everything; scrapes nothing." />
	<LinkCard title="The client (the face)" href="/self-hosting/client/" description="The React frontend your visitors actually see and click." />
	<LinkCard title="The CORS proxy (the edge)" href="/self-hosting/proxy/" description="A free-tier edge relay that carries video bytes so your server doesn't." />
	<LinkCard title="The companion (the familiar)" href="/self-hosting/extension/" description="A tiny browser extension that unlocks the smoothest, most direct playback." />
	<LinkCard title="Your sources (the secret)" href="/self-hosting/sources/" description="Your own private repository of stream providers, bundled at build time." />
</CardGrid>

:::tip[Lumi says]
You do **not** need all five to begin. A backend, a database and the client are
enough to log in and browse. Add the proxy, companion and your sources as you grow.
:::
